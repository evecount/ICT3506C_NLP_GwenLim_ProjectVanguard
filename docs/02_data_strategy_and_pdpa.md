# Data Strategy: Synthetic vs Empirical & PDPA Considerations
*(Original Gemini Recommendation regarding data sourcing)*

You hit on a very core data science debate, but for an academic competency challenge, **using the real 600-message chat history is drastically better** than relying solely on synthetic data. Here is why:

### The "Professor Rubric" Perspective

Synthetic data generated by an LLM tends to be too clean, grammatically uniform, and structurally predictable. The professors evaluating an NLP exemption portfolio want to see how you deal with the real, chaotic nature of human text:

* **The "Super-Individual Secret Club" chat has real noise:** Mixed messaging contexts, short bursts ("Where you", "got food?"), overlapping timestamps, media uploads, and localized syntax (like referencing specific event dates like July 24).
* Cleaning and segmenting *that* specific mess proves you possess actual engineering competency, which completely validates why you should skip the standard module.

---

### The Winning Strategy: Use Both (Hybrid Approach)

Instead of picking one, the gold standard data pipeline architecture uses **both** to build an air-tight submission. You can use your real chat data as the ground truth, and synthetic data to scale it up.

Here is how you structure it:

#### Phase 1: The Core Pipeline (Using the Real Chat)

1. **Export the Data:** Dump the Telegram history into a JSON or CSV format.
2. **Build the Cleaners:** Write the tokenization scripts to parse out the metadata (user handles, timestamps, image references) from the core conversational textual vectors.
3. **Execute the Segmentation:** Run your sliding-window embeddings and cosine similarity checks on the real 600 messages to demonstrate how the model dynamically draws boundary lines between shifting topics (e.g., when the chat shifts from casual food coordination to mapping out the business event on July 24).

#### Phase 2: Evaluation & Evaluation Metrics (Using Synthetic Data)

This is where you make the portfolio undeniable. Since the real chat doesn't come with pre-labeled ground truth headers, it’s hard to calculate a hard mathematical $F_1\text{-score}$ for your text classifier.

1. **Generate Synthetic Benchmarks:** Use an LLM to generate a synthetic dataset of 2,000 group chat messages where you *explicitly know* where the topic switches occur.
2. **Run the Math:** Run your pipeline on the synthetic data to output a formal evaluation suite (Confusion Matrix, Precision, Recall).
